# 📖 Glossaire Python

Le petit dictionnaire pour comprendre les termes bizarres que tu croiseras souvent.

---

### A
- **Argument** : Une valeur que tu passes à une fonction lors de son appel.
- **Async / Await** : Mots-clés utilisés pour la programmation asynchrone (faire plusieurs choses sans bloquer le programme).

### B
- **Built-in** : Désigne une fonction ou un outil déjà présent par défaut dans Python (ex: `print()`, `len()`).

### C
- **Class (Classe)** : Le "moule" ou le plan de fabrication d'un objet en POO.
- **Comprehension (de liste)** : Une syntaxe compacte pour créer une liste à partir d'une autre.

### D
- **Docstring** : Une chaîne de caractères située juste après la définition d'une fonction ou classe pour documenter son usage.
- **Duck Typing** : Philosophie de Python : "Si ça marche comme un canard, ça doit être un canard". On ne vérifie pas le type, mais si l'objet sait faire ce qu'on lui demande.

### E
- **Enumerable** : Un objet dont on peut parcourir les éléments un par un (ex: une liste).

### G
- **Garbage Collector (Ramasse-miettes)** : Système automatique qui libère la mémoire utilisée par des objets dont le programme n'a plus besoin.
- **GIL (Global Interpreter Lock)** : Un mécanisme interne qui empêche plusieurs threads d'exécuter du code Python en même temps (spécifique à CPython).

### I
- **Immutable (Immuable)** : Un objet qu'on ne peut pas modifier après sa création (ex: les Tuples, les Strings).
- **Indentation** : Les espaces en début de ligne. En Python, c'est ce qui définit les blocs de code.

### L
- **Lambda** : Une petite fonction anonyme écrite sur une seule ligne.

### M
- **Method (Méthode)** : Une fonction qui appartient à une classe ou un objet.
- **Module** : Un simple fichier `.py` contenant du code que tu peux importer ailleurs.

### P
- **Package** : Un dossier contenant plusieurs modules Python (fichiers `.py`).
- **PEP (Python Enhancement Proposal)** : Documents officiels décrivant les standards de Python. Le plus connu est **PEP 8** (le guide de style).
- **Pip** : L'outil standard pour installer des bibliothèques Python depuis internet (PyPI).
- **PyPI (Python Package Index)** : Le "magasin" géant gratuit où se trouvent des milliers de bibliothèques Python.

### S
- **Shebang** : La ligne `#!` au début d'un script pour indiquer quel programme doit l'exécuter.

### V
- **Venv (Virtual Environment)** : Un environnement isolé qui permet d'installer des bibliothèques pour un projet sans polluer tout ton ordinateur.
